## 🎯 三個關鍵反作弊邊界(Tony 2026-05-15 提醒)

```
台灣電子發票的客觀現實:
  1. 有發票 → 一定也有小白單(POS 印的明細)
  2. 用載具 → 沒實體發票,但小白單還是會給
  3. 兩張單據(發票 + 小白單)都各自有 "發票號碼" 欄位

潛在攻擊向量:
  客人結帳後手上同時有發票 + 小白單,可能嘗試
  「先掃發票」→ 拿擲骰 → 「再拍小白單」→ 又拿擲骰 = 雙領

系統如何防禦:
  鎖 #3:同 user + 同 table_id 在 20 分鐘內限 1 次(共用 invoices 表)
  → 第二次嘗試會收到「您這一桌已經兌換過一張了」
```
