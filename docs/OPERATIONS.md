## 1. 客人怎麼玩

### A. 第一次來

| 步驟 | 客人動作 | 看到什麼 |
|------|---------|---------|
| 1️⃣ | 用 LINE 掃桌邊立牌 QR | 加 IC 高雄好友 |
| 2️⃣ | 對話框打 `table:XX`(您的桌號代碼) | 收到「綁定成功」訊息 |
| 3️⃣ | 點訊息按鈕進遊戲 | 選一個卡通角色 |
| 4️⃣ | 結帳後三選一拿擲骰機會 | (下面詳細) |
| 5️⃣ | 擲骰 → 走大富翁 → 抽獎 / 拿獎品 | 過 15 點抽大獎 → 下一季開始 |

### B. 第 4 步驟「拿擲骰機會」三條路

```
┌────────────────────────────────────────────────────────┐
│                                                          │
│  📄 路徑 A:電子發票證明聯(紙本)                       │
│     → 客人點 LIFF「📄 掃發票拿擲骰機會」                │
│     → 對準發票左側 QR Code                              │
│     → 2 秒後拿到擲骰機會                                │
│                                                          │
│  📸 路徑 B:用載具(只有小白單)                        │
│     → 客人點 LIFF「📸 我用載具(拍小白單)」            │
│     → 拍小白單                                          │
│     → AI 辨識 3-5 秒後拿到擲骰機會                      │
│                                                          │
│  📷 路徑 C:服務人員的結帳 QR(緊急備援)               │
│     → 服務人員手機開 QR 給客人                          │
│     → 客人 LINE 相機掃                                  │
│     → 立刻拿到擲骰機會                                  │
│                                                          │
└────────────────────────────────────────────────────────┘
```

### C. 重複來店

- **不用重新加 LINE 好友**(永久保留)
- **重新打 `table:XX` 綁桌**(綁定 20 分鐘失效)
- 後面同首次第 4 步
- **已領獎品紀錄永久保留**,不會因下一季消失

### D. 客人卡關

如果客人連續 2 次掃不過,系統會自動跳「🛎️ 請洽現場服務人員」黃色 banner。

→ 客人會主動來找您 → 走第 2 章「結帳 QR」流程補發。

## 2. 服務人員 SOP

### A. 90% 時間什麼都不用做 ✅

- 客人加 LINE 好友:自己掃桌邊 QR
- 客人綁桌:自己打 `table:XX`
- 客人拿擲骰:自己掃發票 / 拍小白單
- 客人玩遊戲:自己擲骰、抽獎

**您只需要在這 3 種狀況介入:**
1. 客人**問怎麼玩** → 教他掃桌邊 QR
2. 客人**卡關**(看到黃色 banner)→ 走結帳 QR 流程
3. 客人**吵說發票沒用過** → 走客訴查詢工具(第 3 章)

### B. 客人問「這遊戲怎麼玩?」

帶客人到桌邊立牌前說:

> 「請您用 LINE 掃這個 QR Code,加我們的 LINE 好友。
> 然後在對話框打 `table:` 加上您的桌號(例如 `table:ZL05`),系統會傳遊戲連結給您。
> 結帳後消費滿 NT$2,000 就能玩擲骰子大富翁。」

### C. 結帳 QR 6 步驟 ⭐(主要流程)

> 客人說「我用載具」、發票忘了帶、或卡關時 → 您主動補發。

**請把這個流程練到 30 秒內完成。**

#### 動作步驟

| 步驟 | 您做的事 |
|------|---------|
| 1️⃣ | 您手機開 `https://ickhh-culinary-game-v2.zeabur.app/admin/checkout` |
| 2️⃣ | 第一次用要輸入 **PIN:`91097496`**(主管會告訴您) |
| 3️⃣ | ⚠️ **輸入發票編號**(從小白單「發票號碼」欄抄,例如 `BM36258896`) |
| 4️⃣ | 輸入消費金額(例如 5000) |
| 5️⃣ | 點「✓ 產生 QR Code」→ 螢幕出現 QR + 倒數 2 分鐘 |
| 6️⃣ | 客人用 LINE 相機掃這個 QR → 完成 |

**下一桌** → 按「下一桌」按鈕清空繼續。

#### 重點規則

- **必須填發票編號** — 同一張發票編號全世界只能用一次(防雙領)
- **如果系統說「⛔ 這張發票已被使用過」** → 系統會顯示之前誰幾時用的 → 直接給客人看
- **NT$2,000 = 1 次擲骰,上限 5 次**(金額 < 2000 系統會擋)
- **QR 2 分鐘失效** — 過期重新產
- **第一個掃的人拿到** — 不要把 QR 截圖傳給多人

#### 發票編號怎麼抄

小白單上有一欄「發票號碼:`BM-36258896`」這樣寫:
- 系統接受 `BM36258896`(無橫線)
- 也接受 `BM-36258896`(有橫線,自動忽略)
- 格式:**2 個英文字母 + 8 個數字**

### D. 客人手機螢幕錯誤訊息對照表

如果客人指著手機螢幕說「掃不過,出現這個訊息」,請對照下表:

| 客人看到的訊息 | 真實原因 | 您怎麼處理 |
|---------------|---------|-----------|
| 「請先掃描桌邊立牌 QR Code 加入 LINE 並綁定桌號」 | 還沒綁桌 | 帶他到桌邊立牌掃 + 對話框打 `table:XX` |
| 「這 QR 不是電子發票格式」 | 客人對到菜單或別的 QR | 引導他對準**發票左側**(不是右側) |
| 「這張發票不是 IC 高雄洲際開的」 | 客人手上是別家(7-11、星巴克)的發票 | 確認手上是 IC 的,若不是 → 沒辦法給擲骰 |
| 「發票日期過期,只能使用今天/昨天開立的發票」 | 客人翻出舊發票 | 走 **結帳 QR 補發**(因為他今天確實消費了) |
| 「此發票已被使用過」 | 已經領過,或同張掃兩次 | 確認客人沒重複領 → 若客人堅持沒領過 → 走 **客訴查詢工具**(第 3 章) |
| 「您這一桌已經兌換過一張發票了」 | 同桌已用一張(可能同行人領了) | 確認後告知;若真的沒領 → 走 **結帳 QR 補發** |
| 「金額需滿 NT$2,000」 | 消費沒到門檻 | 告知客人:「您消費還沒到 NT$2,000 參加門檻」 |
| 「圖片不夠清楚,請重拍」(拍小白單時) | 光線不夠 / 對焦失敗 | 引導到光線好的位置重拍,或走 **結帳 QR** |
| 「這張不像是 IC 高雄洲際的單據」 | AI 看不出 IC 抬頭 | 確認小白單完整 → 走 **結帳 QR** |
| 黃色「🛎️ 已嘗試 2 次仍無法兌換」 | 連續失敗 2 次 | 直接走 **結帳 QR** |

**結論:幾乎所有狀況的最終答案 = 走結帳 QR 流程補發。**

### E. 我絕對不要做什麼 ❌

- ❌ **不要把 PIN 寫在紙上貼牆上**(口頭傳達就好)
- ❌ **不要把 QR Code 截圖傳給多人**(只第一個掃的拿得到)
- ❌ **不要為了討好客人破壞規則**(同桌一張是設計上的反作弊)
- ❌ **不要替客人輸 LINE 帳號 / 密碼**(他自己掃就好)
- ❌ **不要相信「我朋友剛剛已經幫我領了」這種說法**(查詢工具會驗證)

## 3. 客訴怎麼處理 ⭐

### 情境:客人吵「我這張發票明明沒用過!」

不要用嘴跟客人爭執,**用系統紀錄說話**。

#### 步驟

1. **您手機開** `https://ickhh-culinary-game-v2.zeabur.app/admin/lookup`
2. **輸入 PIN**(同結帳 QR,通常已記住)
3. **輸入客人發票編號**(例如 `BM36258896`)
4. **點查詢** → 看三種結果:

#### 結果 1:🟢 此發票尚未使用

**畫面綠色**,寫「此發票尚未使用」。

→ 客人沒說謊。引導他自助掃發票,或為他開結帳 QR。

#### 結果 2:🔴 此發票已被使用

**畫面紅色**,顯示:
- 使用時間(精確到分鐘)
- 使用者 LINE 暱稱
- 桌號
- 當時金額
- 給予的擲骰次數
- 兌換方式(掃發票 / 拍小白單 / 服務人員結帳 QR)

**把這個畫面給客人看**。
- 紀錄無法竄改
- 客人通常會記起來「啊,我同事幫我領了」或「我上次來領過了」

#### 結果 3:🟡 已開 QR 但客人沒掃

**畫面黃色**,顯示剛剛 2 分鐘內有人開過 QR 但客人還沒掃。

→ 請客人立刻掃,或等 2 分鐘後 QR 過期再重新開。

## 5. IT / 行銷主管

### A. 看客戶數據

開 `https://ickhh-culinary-game-v2.zeabur.app/admin/customers`(需 LINE 登入 + staff_whitelist)

可以看到:
- 客戶總數、本月新增
- 5 餐廳熱度排行
- 個別客戶 timeline
- CSV 下載

### B. 加 staff 白名單(讓某 LINE 帳號可進 /admin/customers)

```bash
curl -X POST https://ickhh-culinary-api-v2.zeabur.app/api/admin/staff \
  -H "X-Admin-Token: <ADMIN_TOKEN>" \
  -H "Content-Type: application/json" \
  -d '{"user_id": "U1234...", "name": "張三", "role": "super_admin"}'
```

ADMIN_TOKEN 存在 Zeabur backend 環境變數,只有 IT 知道。

### C. 換 PIN

1. 開 Zeabur dashboard → culinary-journey-map → backend 服務 → 環境變數
2. 找 `STAFF_NUMERIC_PASSWORD` → 編輯 → 改新值 → 儲存
3. 自動 redeploy ~30 秒
4. 通知餐飲部主管新 PIN

### D. 換 OpenAI 模型(AI 視覺辨識)

同 C,但改 `OPENAI_VISION_MODEL`:
- `gpt-4o`(預設,中文 OCR 準確)
- `gpt-4o-mini`(便宜 10 倍,小字偶爾出錯)
- `gpt-5` / `gpt-5-mini`(如可用)

### E. 看 debug 日誌

Zeabur backend 服務 → 「日誌」分頁,搜尋:
- `[invoice/redeem]` 掃發票
- `[invoice/redeem-receipt]` 拍小白單
- `[receiptVision]` OpenAI vision 呼叫
- `[checkout-ticket/issue]` 服務人員開 QR
- `[staff-lookup]` 客訴查詢

## 6. 應變措施

### 異常 1:Backend 全掛(所有客人都掃不過)

**症狀:** `https://ickhh-culinary-api-v2.zeabur.app/api/health` 不回 200

**現場:** 餐廳保留客人發票,記下姓名 + LINE,告知「系統維護中,請稍後再試」
**IT:** Zeabur 服務狀態 → 重啟 → 5 分鐘修不好進「全人工模式」(異常 8)

### 異常 2:Frontend 掛(網頁打不開)

**症狀:** LINE 連結點開白屏 / 404

**IT:**
1. 確認是不是 Zeabur frontend 服務
2. 用備用 URL `ickhh-culinary-game-v2.zeabur.app`
3. 看最近 deploy,rollback 到前一版

### 異常 3:OpenAI 掛(小白單拍照全失敗)

**症狀:** 客人看到「拍照辨識暫時無法使用」

**IT:**
1. 檢查 OpenAI 帳號餘額 / API key
2. 短期:Zeabur 設 `OPENAI_API_KEY=disabled` → 客人會被引導去找服務人員
3. 服務人員走「結帳 QR」流程救援(不依賴 OpenAI)

### 異常 4:資料庫掛

**症狀:** backend 啟動失敗 / `[db] Postgres smoke failed`

**IT:** Zeabur 看 postgresql 服務,重啟。**千萬不要 drop 任何 table** — 客戶資料、發票紀錄全在裡面。

### 異常 5:PIN 外洩

**症狀:** 行銷後台看到大量無消費的擲骰機會,或服務人員回報「我沒開 QR 但客人說已經有擲骰」

**處理:**
1. IT 立刻在 Zeabur 改 PIN
2. 主管口頭告知新 PIN
3. 行銷主管統計損失

### 異常 6:同行人也想領

**症狀:** 客人 A 領完,同行 B 也想領,但同桌系統擋

**現場:** 告知「每桌每次用餐限一張,下次自己一桌時再玩」。
- **不要破壞規則** — 這是反作弊設計
- 若兩人是真的不同桌(罕見)→ 服務人員可開兩張結帳 QR 各別給

### 異常 7:客人手機 / LINE 問題

**症狀:** 「掃不到 QR」「LINE 點不開」

**現場:**
1. 確認相機沒髒
2. 確認 LINE 是最新版
3. 連飯店 WiFi
4. 真的不行 → 結帳 QR 備援(他至少有 LINE 帳號就好)

### 異常 8:全人工模式(系統掛 30 分鐘以上)

**情境:** Backend 掛太久 / Demo 中 / 重要客人在場

**現場:**
1. 服務人員給「紙本兌換券」(餐飲部準備 100 張備用)
2. 兌換券寫:日期 / 桌號 / 消費金額 / 客人 LINE 暱稱
3. 客人下次來時憑券補發擲骰(IT 開 admin/checkout 補)

### 異常 9:Demo 當天 / VIP 在場

**事前:**
- 5/30 前一天測試所有路徑
- 5/31 demo 主管駐場
- IT 開 Zeabur 監控頁面盯緊

**現場:**
- 多備 5 部服務手機,預先登入 `/admin/checkout` 並輸過 PIN
- 行銷主管手機備用
- 客人卡關不超過 1 分鐘就介入

### 異常 10:服務人員當場不會操作

**事前:** 服務站抽屜放小卡(見第 4-B)

**現場:** 找主管或 LINE 群組求援。**不要當場跟客人試錯。**

### 異常 11:客人作弊 / 用假發票

系統會自動擋:
- 偽造 QR → 賣方統編 ≠ 91097496 → 擋
- 舊發票截圖 → 日期不對 → 擋
- 同張發票多次 → invoice_no PK → 擋
- Photoshop 改金額 → 客人在發票上動手腳,法律歸 IC 一邊

**重大作弊事件:**
1. 行銷主管在 /admin/lookup 截圖存證
2. 涉刑(偽造文書)走法務流程

## 7. 反作弊規則

服務人員不用記,但**理解為什麼系統會擋人**,跟客人解釋時比較有底氣:

| 規則 | 系統怎麼擋 | 客人會看到 |
|------|----------|----------|
| 發票必須是 IC 高雄開的 | 檢查賣方統編 `91097496` | 「這張發票不是 IC 高雄洲際開的」 |
| 發票必須是今天或昨天 | 比對日期 | 「發票日期過期,只能使用今天/昨天的發票」 |
| 同一桌限一張 | 同 user + 同 table_id 20 分鐘內限一次 | 「您這一桌已經兌換過一張了」 |
| 全域唯一 | invoice_no PRIMARY KEY 全域唯一 | 「此發票已被使用過」 |
| 金額門檻 | total ≥ NT$2,000 | 「金額需滿 NT$2,000」 |
| 綁桌時效 | 必須 20 分鐘內綁過桌 | 「請先掃描桌邊立牌 QR」 |
| 小白單必須是 IC 的 | AI 視覺辨識 IC 抬頭 | 「這張不像是 IC 高雄洲際的單據」 |
| 小白單照片要清楚 | gpt-4o vision confidence ≥ 0.85 | 「圖片不夠清楚,請重拍」 |

## 8. 常見問答 FAQ

**Q1. 消費 NT$3,500 應該幾次擲骰?**
A:**1 次**。系統公式 `floor(amount / 2000)` = `floor(3500 / 2000)` = 1。

**Q2. 消費 NT$15,000?**
A:**5 次**(系統上限)。即使 15000/2000=7.5 也只給 5。

**Q3. 客人分兩次結帳,兩張發票可不可以分別掃?**
A:**不行**。同一桌 20 分鐘內限一張。請客人合併結帳。

**Q4. 同行兩人各有 LINE 可不可以各拿一份?**
A:**不行**,還是同桌限一張。若要兩人都玩,得分兩桌坐。

**Q5. 客人加了 LINE 但沒玩,過幾天回來還算他嗎?**
A:**算**。LINE 好友身份永久,只是 table_binding 失效(20 分鐘 TTL)。下次來重新打 `table:XX` 即可。

**Q6. 過 15 點抽到大獎後,獎品紀錄會消失嗎?**
A:**不會**。`earned_rewards` 永久保留。客人下一季從 0 點開始,但獎品歷史一直在。

**Q7. 結帳 QR 失效後再產一張,前一張就完全沒用嗎?**
A:**沒用**。每個 token 有獨立 expires_at + used_at,過期或被掃過都不能再用。

**Q8. 管理員可以「補發」擲骰給特定客人嗎?**
A:沒有直接 UI,要走結帳 QR 流程。或 IT 直接寫 SQL(技術手段,留紀錄)。

**Q9. 結帳 QR 顯示「金額需滿 NT$2,000」?**
A:客人消費沒到 2000,告知「您消費 NT$X 還沒到參加門檻 NT$2,000,下次再來玩」。

**Q10. 同一張結帳 QR 兩個客人同時掃?**
A:第一個掃的人拿到,第二個會看「此 QR Code 已被使用過」。服務人員為第二人重產一張即可。

**Q11. 客人手上有電子發票證明聯,系統卻顯示已使用,怎麼辦?**
A:走 `/admin/lookup` 查詢工具。99% 是客人自己忘了已領,或同行人幫他領了。把使用紀錄畫面給客人看。

**Q12. 客人說「服務人員昨天答應我可以領,你怎麼擋我?」**
A:走 `/admin/lookup` 確認該發票編號真實狀態。系統紀錄不可改,以系統為準。若真的是系統 bug,聯絡 IT。
